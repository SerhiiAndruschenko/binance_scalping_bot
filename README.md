# Binance Scalping Bot

Скальпінговий торговий бот для Binance USDT-M Futures.
Самодостатній проєкт — не залежить від жодних спільних файлів з основним ботом.

---

## Ключові відмінності від основного бота

| Параметр        | Основний         | Скальпінг          |
|-----------------|------------------|--------------------|
| Таймфрейм       | 15m              | **1m**             |
| Скан            | кожні 60 сек     | **кожні 15 сек**   |
| TP / SL         | 1.5% / 0.8%      | **0.3% / 0.2%**    |
| Плече           | x5               | **x3**             |
| Баланс          | $1 000           | **$300**           |
| Індикатори      | EMA + RSI + MACD | **EMA + RSI + VWAP + OrderBook** |
| Угод / день     | 3–6              | **15–30**          |
| Макс позицій    | 3                | **2**              |
| Ризик / угоду   | 2%               | **1%**             |
| Денний ліміт    | -5%              | **-3%**            |

---

## Стратегія входу

**LONG** — всі 4 умови одночасно:
1. EMA9 > EMA21 (висхідний тренд)
2. Ціна ≤ VWAP (є простір для руху вгору)
3. RSI між 35 і 65 (не перекуплено/перепродано)
4. Order Book Imbalance > 60% BID (покупці переважають)

**SHORT** — всі 4 умови одночасно:
1. EMA9 < EMA21 (низхідний тренд)
2. Ціна ≥ VWAP (є простір для руху вниз)
3. RSI між 35 і 65
4. Order Book Imbalance > 60% ASK (продавці переважають)

## Встановлення

```bash
git clone <repo-url> binance-scalping-bot
cd binance-scalping-bot

# Копіюємо та заповнюємо .env
cp .env.example .env

# Встановлюємо залежності
pip install -r requirements.txt
```

---

## Налаштування `.env`

```
API_KEY=<Binance API key>
API_SECRET=<Binance API secret>
TELEGRAM_BOT_TOKEN=<токен Telegram бота>
TELEGRAM_CHAT_ID=<ID чату>
TELEGRAM_THREAD_ID=<ID топіку — залишити пустим якщо не потрібно>
MAX_TRADING_BALANCE=300
MAX_OPEN_TRADES_GLOBAL=2
STATE_DIR=.
```

> **Testnet**: перемкніть `TESTNET = True` → `False` у `config.py` для реальної торгівлі.

---

## Запуск

```bash
# Завантажуємо .env автоматично через python-dotenv
python main.py
```

На Railway або іншому сервері встановіть ENV-змінні через панель керування.
Файл стану `scalp_state.json` зберігається в `STATE_DIR` (за замовчуванням — поточна директорія).

## Telegram команди

Всі команди мають префікс `/s_` щоб не конфліктувати з основним ботом.

| Команда     | Дія                                    |
|-------------|----------------------------------------|
| `/s_status` | Поточний статус бота і відкриті позиції |
| `/s_today`  | Денна статистика (P&L, угоди, winrate) |
| `/s_month`  | Місячна статистика                     |
| `/s_pause`  | Призупинити відкриття нових угод       |
| `/s_resume` | Відновити торгівлю                     |
| `/s_stop`   | Graceful shutdown з закриттям позицій  |

Всі повідомлення бота починаються з префіксу `[SCALP]` —
це дозволяє легко розрізняти сповіщення від скальпінгового та основного ботів в одному чаті або топіку.

## Структура проєкту

```
binance-scalping-bot/
├── main.py             # Точка входу, основний цикл (15 сек)
├── config.py           # Всі параметри і ENV змінні
├── binance_client.py   # Обгортка над python-binance (Futures API)
├── strategy.py         # Логіка сигналів: EMA + RSI + VWAP + OrderBook
├── trader.py           # Відкриття/закриття позицій, SL/TP моніторинг
├── risk_manager.py     # Розмір позиції, денний ліміт, персистентність
├── notifications.py    # Telegram сповіщення
├── telegram_bot.py     # Telegram команди (/status, /pause тощо)
├── logger.py           # Логування (scalping_bot)
├── scalp_state.json    # Стан бота (створюється автоматично)
├── .env                # Секрети (не комітити!)
├── .env.example        # Шаблон
└── requirements.txt
```
